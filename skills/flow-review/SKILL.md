# 心流规律分析

读取 `~/.flow-data/logs.json` 中的所有记录，进行深度分析。

如果文件不存在或记录少于 10 条，提示用户先用 `/flow-state:flow-log` 积累更多数据。

如果 `$ARGUMENTS` 为 `week`，只分析最近 7 天的数据；`all` 或不传参数则分析全部数据。

---

## 分析维度

读取完数据后，按以下结构逐步分析并展示结果：

### 一、数据概览

```
分析区间：[开始日期] 至 [结束日期]
总记录数：X 条
平均每天：X 条
心流分布：是 X% / 部分 X% / 否 X%
平均专注度：X.X / 10
平均情绪值：X.X / 10
```

---

### 二、哪些活动最容易触发心流？

将所有记录按活动分组，计算每类活动的：
- 心流率（"是" 的比例）
- 平均专注度
- 平均情绪值
- 出现次数

**展示格式（按心流率排序）：**
```
排名  活动          心流率   专注度   情绪   次数
1     [活动名]      85%     8.5     8.2    7次
2     [活动名]      60%     7.2     7.8    5次
...
```

**解读：** 指出心流率最高的 2～3 个活动，说明这些活动的共同特征（如：独立完成、有明确目标、需要技能匹配等）。

---

### 三、一天中哪些时段更容易进入心流？

将记录按小时分组（或分为上午/下午/傍晚/夜间 4 段），计算各时段的心流率和专注度。

**展示格式（热力图文字版）：**
```
时段         心流率   专注度   记录数
06-09        —%       —        —条
09-12        —%       —        —条
12-15        —%       —        —条
15-18        —%       —        —条
18-21        —%       —        —条
21-24        —%       —        —条
```

**解读：** 指出你的心流高峰时段，以及低谷时段，并给出具体建议（例如："你的专注高峰在 9-11 点，建议把最重要的创作类工作安排在这个时段"）。

---

### 四、环境因素分析

分析地点和陪伴者与心流的关系：

**按地点分组：**
```
地点      心流率   专注度   次数
[地点1]   —%       —        —
...
```

**按陪伴者分组：**
```
陪伴情况   心流率   专注度   次数
独自一人   —%       —        —
...
```

**解读：** 说明哪种环境组合对你最有利。

---

### 五、低效时段分析

找出心流率最低、专注度最低的时段和活动，分析可能原因（如：固定时间段、特定活动类型、特定地点）。

---

### 六、个性化行动建议

基于以上分析，给出 3 条具体的、可操作的建议：

1. **增加心流的建议**：基于高心流活动的特征，建议如何重新设计日程或工作方式
2. **时间编排建议**：基于高峰时段，建议如何重新安排一天的活动顺序
3. **减少低效时间的建议**：针对低心流的时段和活动，给出具体改进方法

---

### 七、还需要更多数据吗？

告知用户目前的数据量是否足够（建议至少 40 条以上才能得出可靠规律），以及还需要追踪多少天。

如果数据充足（≥40 条），鼓励用户开始实施建议，并在 2 周后再次运行 `/flow-state:flow-review` 观察变化。
