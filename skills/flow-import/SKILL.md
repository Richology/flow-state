# 导入手机记录

用户会粘贴一段或多段在手机上随手记的内容，可能是口语化的文字，可能格式不完整，可能只有一句话。

## 第一步：解析内容

从用户发来的文字中，尽力提取以下字段：

- **timestamp**：时间。可能是「下午」「3点」「刚才」「今天上午」这类表述，结合当前时间推算为具体时间，格式 `YYYY-MM-DDTHH:MM:00`。无法推断则留空 `""`
- **activity**：在做什么。提取核心活动词，如「写方案」「开会」「跑步」「发呆」
- **location**：在哪里。如「咖啡馆」「家里」「公司」「地铁上」。无法判断则留空
- **with_whom**：和谁在一起。如「独自一人」「和同事」「和朋友」。无法判断则留空
- **focus**：专注程度（1-10）。从描述中推断，如「很投入」→ 8-9，「心不在焉」→ 2-3，「还行」→ 5-6。无法判断则留空 `null`
- **mood**：情绪状态（1-10）。从描述中推断，如「很愉快」→ 8-9，「有点烦」→ 3-4。无法判断则留空 `null`
- **flow**：心流状态。从描述推断 `"yes"` / `"partial"` / `"no"`。无法判断则留空 `""`
- **note**：用户原始文字原样保留

**容错原则：能提取多少就提取多少，不因为信息不完整而拒绝或报错。**

---

## 第二步：展示解析结果，请用户确认

用清晰易读的格式展示解析出的内容，例如：

```
我帮你解析了这条记录：

  时间：今天 15:30
  活动：写方案
  地点：咖啡馆
  陪伴：独自一人
  专注：8/10
  情绪：7/10（从「感觉不错」推断）
  心流：是

有几个地方我不确定，帮我确认一下：
  · 情绪 7 分准确吗？
```

只对真正不确定的字段提问，不要把所有字段都列出来让用户逐一确认——那会让人觉得麻烦。

如果有多条记录，逐条展示，统一确认。

---

## 第三步：保存到数据文件

用户确认后，将记录追加保存到 `~/.flow-data/logs.json`。

如果文件不存在，先创建（内容为空数组 `[]`）。

保存时在每条记录里加一个字段 `"source": "import"` 以区分来源。

---

## 第四步：给一条具体的改进提示

**每次保存完之后，只给一条提示**，针对这次记录里最关键的缺失信息，给出下次可以直接套用的句子。

提示要具体，不要泛泛而谈：

- 缺时间 → 「下次可以这样记：『下午3点在咖啡馆写方案，很投入』，加上时间点，我就能分析你哪个时段最容易进入状态。」
- 缺地点/陪伴 → 「下次加上在哪、一个人还是有人陪，比如『在咖啡馆独自写方案』，这样我能看出环境对你的影响。」
- 缺状态描述 → 「下次加一两个词描述感受，比如『很投入』『有点分心』『感觉还行』，我会帮你换算成分数。」
- 记录已经很完整 → 「这条记录很完整，继续保持就好。」

**不要在一次提示里说多件事。** 选最重要的一条说。

---

## 完成提示

```
✓ 已保存 X 条记录，累计共 Y 条。

[一条具体的改进提示]
```

---

## 附：用户可以发什么样的内容

这个 skill 能处理各种格式，比如：

- 「下午在咖啡馆写东西，很投入」
- 「3点 开会 公司 无聊死了」
- 「刚跑完步，一个人，脑子很清醒，好久没这种感觉了」
- 「今天上午10点，在家看书，孩子在旁边，有点分心，但大体还行」
- 多条合并发送，用换行隔开

越写越完整是目标，但从一句话开始就够了。
