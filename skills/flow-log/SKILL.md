# 心流打卡 — 记录当下状态

铃响了，现在花 30 秒记录一下当下的状态。

## 记录流程

请按顺序问用户以下问题，每个问题单独提问，等用户回答后再问下一个：

**问题 1：你现在在做什么？**
（一句话描述当前活动，例如：写代码、开会、刷视频、和朋友聊天、发呆）

**问题 2：你在哪里？和谁在一起？**
（例如：家里/独自一人、办公室/同事在旁、咖啡馆/陌生人）

**问题 3：专注程度如何？（1-10）**
- 1-3：完全心不在焉，思绪游离
- 4-6：勉强在做，但注意力不集中
- 7-8：比较投入
- 9-10：完全沉浸，外界干扰不到我

**问题 4：此刻情绪状态如何？（1-10）**
- 1-3：负面，烦躁/焦虑/无聊/疲惫
- 4-6：一般，平淡
- 7-8：积极，愉悦/平静
- 9-10：非常好，兴奋/满足/充实

**问题 5：你觉得自己现在处于心流状态吗？**
回答选项：
- **是** — 时间飞逝，完全忘我，做这件事感觉自然而然
- **部分** — 有点投入，但还没完全进入状态
- **否** — 明显没有，心不在焉或被迫在做

**可选问题 6：还有什么想备注的吗？**
（如：被打断了、刚睡醒、刚吃饱、有截止日期压力等）

## 保存记录

收集完所有回答后，将记录追加保存到 `~/.flow-data/logs.json` 文件中。

**数据格式如下：**
```json
{
  "timestamp": "2026-04-18T14:32:00",
  "activity": "用户回答的活动",
  "location": "位置",
  "with_whom": "陪伴者",
  "focus": 8,
  "mood": 7,
  "flow": "yes|partial|no",
  "note": "备注（可选）"
}
```

如果 `~/.flow-data/logs.json` 不存在，先创建文件（内容为空数组 `[]`），再追加新记录。

保存成功后，显示一条简短的确认消息，例如：
```
✓ 已记录 [时间] — [活动] — 心流：[是/部分/否]
本周已记录 X 次，目标 40 次。继续加油！
```

如果用户在命令后面附带了参数（`$ARGUMENTS`），将其作为备注直接填入 note 字段，跳过问题 6。
